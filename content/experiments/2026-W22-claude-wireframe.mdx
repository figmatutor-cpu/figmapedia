## 실험 배경

요즘 주니어 디자이너 사이에서 "AI가 와이어프레임도 그려주냐"는 질문이 부쩍 늘었습니다.
실제로 Claude의 최근 업데이트에는 캔버스 도구가 들어왔고, 텍스트로 화면 구조를 설명하면
간단한 와이어를 그려줍니다. 정말 쓸 만한지, 어디까지 가능한지 직접 실험해봤습니다.

## 시나리오

- **앱 종류**: 음식 배달 앱
- **요청 화면**: 홈, 메뉴 상세, 주문서, 결제 완료 — 총 4개
- **비교군**: ChatGPT-4o, Figma AI, 직접 손그림

## 프롬프트 셋업

```text
역할: 시니어 모바일 UX 디자이너
출력: 4개 화면의 와이어프레임 (저충실도)
제약: 8pt 그리드, 한국 모바일 디자인 가이드 준수
```

## 단계별 결과

### 1단계: 홈 화면

Claude가 가장 잘 한 부분. 카테고리 그리드 + 추천 메뉴 카드 + 하단 탭바를
모두 합리적인 위치에 배치했습니다. 다만 검색바를 상단 고정으로 가정한 점은
한국 배달앱 관행과 일치합니다.

### 2단계: 메뉴 상세

여기서부터 한계가 보입니다. **옵션 선택 UI**(예: 사이드 메뉴 추가, 사이즈)는
체크박스/라디오 구분 없이 단순 리스트로 그려졌습니다. 인터랙션 의도를 추가로
지정해야 했습니다.

### 3단계: 주문서

가장 흥미로운 발견. Claude는 **배달비, 할인 적용, 최종 금액**의 시각적
위계까지 자동으로 잡았습니다. 다만 약관 동의 영역을 빠뜨렸는데, 한국 앱에서는
법적으로 필요한 요소라 사람이 채워줘야 합니다.

### 4단계: 결제 완료

가장 평범한 결과. 체크 아이콘 + 주문번호 + 예상 시간 — 정석이지만 차별점은 없습니다.
