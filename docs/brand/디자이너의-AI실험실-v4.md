**2\. 차별화 전략**

**2-1. 경쟁 플랫폼 비교**

| 구분          | 인프런             | 숨고               | 퍼스트펌귄 (우리)             |
| :------------ | :----------------- | :----------------- | :---------------------------- |
| 파는 것       | 정리된 과거의 지식 | 일회성 전문가 매칭 | 지금 이 순간의 실험과 축적    |
| 콘텐츠 속도   | 제작에 수개월      | 해당 없음          | 이번 주 기능을 이번 주 실험   |
| 주제 선정     | 운영팀 일방적 기획 | 해당 없음          | 멤버 투표로 선정 (Arena 방식) |
| 관계          | 강사→수강생        | 거래 종료시 끝     | 실험 파트너 \+ 라이브 발표자  |
| 네트워크 효과 | 약함               | 없음               | 멤버가 늘수록 DB가 풍부해짐   |

**2-2. 핵심 차별화: 실험 → 기록 → 축적 시스템**

멤버들이 쌓는 실험 기록이 곧 커뮤니티의 자산이 됩니다. 시간이 갈수록 경쟁자가 따라올 수 없는 진입장벽이 형성됩니다.

| 요소                | 내용                                                           | 차별점                            |
| :------------------ | :------------------------------------------------------------- | :-------------------------------- |
| 실험 아카이브 DB    | 도구명 · 실험내용 · 결과 · 적용가능성을 태그로 정리, 검색 가능 | 멤버 증가 \= DB 풍부 \= 가치 상승 |
| 주간 실험 리포트    | 멤버 결과물을 정리해 매주 발행                                 | 요약본 블로그 공개로 SEO 유입     |
| 멘토 \= 실험 파트너 | “나도 해봤는데 비교해보자” 톤의 쌍방향 관계                    | 실제 활동 이력 기반 신뢰도        |

**3\. 주간 실험 싸이클**

매주 반복되는 ‘투표 → 실험 → 투표 → 라이브’ 순환이 커뮤니티의 심장박동입니다.

| MON | 주제 투표 오픈 운영팀이 후보 3\~4개를 올립니다. 멤버 제안 주제 중 추려서 올리거나, 운영팀이 직접 기획합니다. 디스코드 리액션 투표로 간편하게 진행합니다. |
| :-: | :------------------------------------------------------------------------------------------------------------------------------------------------------- |

| TUE | 투표 마감 \+ 실험 가이드 발행 1위 주제 발표. 운영팀이 간단한 실험 가이드(배경, 시도해볼 포인트, 공유 템플릿)을 올립니다. |
| :-: | :----------------------------------------------------------------------------------------------------------------------- |

| WED\~FRI | 멤버 실험 \+ 결과 공유 멤버들이 각자 실험하고 \#주간-실험 채널에 결과물을 공유합니다. 스크린샷, 파일, 후기 등 자유 형식으로 올립니다. |
| :------: | :------------------------------------------------------------------------------------------------------------------------------------ |

| SAT | 베스트 실험 투표 공유된 결과물 중 ‘가장 인사이트 있는 실험’에 투표합니다. 1위는 다음 주 라이브 발표자로 선정됩니다. |
| :-: | :------------------------------------------------------------------------------------------------------------------ |

| SUN\~MON | 리포트 발행 \+ 라이브 준비 운영팀이 결과물을 정리해 실험 리포트로 발행하고 아카이브 DB에 등록합니다. 투표 1위 멤버와 라이브 세션을 준비합니다. |
| :------: | :--------------------------------------------------------------------------------------------------------------------------------------------- |

**3-1. 라이브 세션 설계**

투표 1위 멤버가 직접 자신의 실험 과정과 결과를 발표하는 월 2\~4회 라이브 세션입니다.

| 항목          | 내용                                                       |
| :------------ | :--------------------------------------------------------- |
| 발표자        | 해당 주 베스트 실험 투표 1위 멤버                          |
| 형식          | 30분 발표 \+ 15분 Q\&A (온라인, 디스코드 스테이지)         |
| 발표 내용     | 실험 배경 → 과정 → 결과 → 실무 적용 팁                     |
| 다시보기      | VOD로 웹사이트에 멤버 전용 공개                            |
| 발표자 리워드 | 다음 달 멤버십 무료 \+ 멘토 프로필에 ‘발표 이력’ 뱃지 부여 |

**3-2. 게임화 요소**

| 요소             | 설명                                               | 효과                     |
| :--------------- | :------------------------------------------------- | :----------------------- |
| 토픽 메이커 뱃지 | 제안한 주제가 투표로 채택되면 뱃지 부여            | 주제 제안 동기 부여      |
| 실험실 MVP       | 베스트 실험 투표 1위 선정 시 뱃지 \+ 라이브 발표권 | 질 높은 결과물 공유 유도 |
| 연속 참여 뱃지   | 4주 연속 실험 참여 시 특별 뱃지                    | 꾸준한 참여 유도         |
| 발표자 리워드    | 라이브 발표 시 다음 달 멤버십 무료                 | 발표 참여 동기           |
| 멘토 승급        | 발표 3회 이상 시 멘토 자격 부여                    | 멤버 → 멘토 전환 경로    |

| 순환의 핵심 투표 → 실험 → 투표 → 라이브 → 아카이브 이 순환이 매주 반복되면 세 가지가 동시에 일어납니다. 1\) 멤버들이 매주 돌아올 이유가 생깁니다 (Retention) 2\) 실험 아카이브가 자동으로 쌓입니다 (Asset) 3\) 멤버가 멘토로 성장하는 경로가 열립니다 (Growth) |
| :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
