**1\. v1 → v2 변경사항 요약**

Figmapedia(figmapedia.com) 통합으로 인한 주요 변경점을 정리합니다.

| 항목        | v1 (PRD 초판)     | v2 (Figmapedia 통합)                   |
| :---------- | :---------------- | :------------------------------------- |
| 웹 플랫폼   | Framer 신규 제작  | figmapedia.com에 페이지 추가 (Next.js) |
| 백엔드      | 없음 (Notion DB)  | Supabase (DB \+ Auth \+ Storage)       |
| 아카이브    | Notion DB 임베드  | 프론트 프론트 프론트 DB 직접 구축      |
| 인증        | Discord 역할만    | Supabase Auth \+ Discord 연동          |
| 콘텐츠 퍼널 | 없음 (SNS 유입만) | 프롬프트피디아 → AI실험실 전환 CTA     |
| SEO 자산    | 제로부터 시작     | 기존 피그마 키워드 SEO 계승            |
| 멘토 프로필 | 별도 페이지       | 기존 팀원 소개 확장형                  |
| 커뮤니티    | 별도 문의         | 기존 /community 확장                   |

| 통합의 핵심 이점 1\. SEO 자산 계승: 피그마 키워드로 이미 쌍인 검색 유입을 그대로 활용 2\. 콘텐츠 퍼널: 프롬프트 피디아 45개 → AI 실험실 전환 경로 3\. 기술 재활용: Supabase(DB+Auth) \+ Next.js 그대로 활용 4\. 개발비 절감: Framer 신규 제작 불필요, 페이지 추가만 5\. 브랜드 신뢰: ‘Figmapedia’라는 기존 브랜드에 실험실 서브브랜드 추가 |
| :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

**7\. 자동화 플로우**

**7-1. 결제 → 온보딩**

1. 사용자가 /membership에서 Stripe Checkout으로 결제

2. Stripe webhook (checkout.session.completed) → Supabase Edge Function

3. Supabase: users 테이블에 role='member' \+ subscription_id 저장

4. Zapier: Stripe → Discord ‘Member’ 역할 부여

5. Zapier: Stibee 리스트 추가 \+ 웰컴 이메일 발송

**7-2. 해지 플로우**

6. Stripe webhook (customer.subscription.deleted) → Supabase Edge Function

7. Supabase: users.role → 'free' 변경

8. Zapier: Discord ‘Member’ 역할 제거 \+ Stibee 리스트 제거
